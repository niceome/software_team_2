# 서비스 : TableMate

## Vision Statement

프로젝트 소개

대학생들은 종종 다음과 같은 상황을 경험합니다.
	- 혼자 점심을 먹기 애매할 때
	- 새로운 사람들과 교류하고 싶을 때
	- 같은 학교 학생들과 식사 약속을 잡고 싶을 때
	

타겟 사용자
	- 캠퍼스에서 점심을 혼자 먹는 경우가 많은 대학생
	- 비슷한 음식 취향을 가진 새로운 사람을 만나고 싶은 학생
	- 식사 시간에 가벼운 사회적 교류를 원하는 학생


TableMate는 이러한 문제를 해결하기 위해 만들어진 서비스입니다.

사용자의 선호 음식과 식사 시간대를 기반으로 연결해 줌으로써 혼자 식사하는 상황에서의 어색함을 줄이고 캠퍼스 내 자연스러운 교류를 유도합니다.


## Project Goals & Scope

비지니스 목표
	- 사용자, 게시글, 음식 취향을 관리할 수 있는 백엔드 시스템을 개발한다.
	- CRUD 기반의 게시판 매칭 시스템을 구현한다.
	- 서버와 클라이언트간의 실시간 데이터 송수신 기능을 위하여 웹소켓을 도입하여 구현한다.
	- 음식 취향 기반으로 점심 친구를 찾을 수 있는 동작 가능한 프로토타입을 완성한다.


 주요 기능
 	- 사용자의 회원가입 및 로그인 시스템
	- 선호 음식 선택 기능
	- 점심 식사 모집 게시글 작성을 통한 함께 식사할 사람 모집
	- 게시글 관리 시스템
	- 음식 취향 기반 게시글 필터링 기능
	- 댓글 또는 참여 관리

Out of Scope
	- 사용자 간 리뷰 및 평가 시스템
	- 모바일 어플리케이션 버전


## Stakeholders & Users

Stakeholders
	- 개발자 : 시스템 설계 및 구현을 담당하는 소프트웨어공학론 2팀
	- 교수님 : 프로젝트 진행을 지도하고 결과를 평가하는 교수님
	- 사용자 : 점심 친구를 찾기 위해 서비스를 이용하는 대학생들
	- 테스터 : 개발 과정에서 시스템 기능을 테스트하는 팀 구성원

Stakeholders's Requirement
	- 개발자 : 명확한 백엔드 시스템 아키텍처, 잘 정의된 DB구조와 API 설계, 유지보수가 쉬운 모듈형 코드 구조
	- 교수님 : 시스템 설계 및 CRUD 기능 확인, 체계적인 프로젝트 문서화
	- 사용자 : 점심 친구를 쉽게 찾을 수 있는 간단한 서비스, 음식 취향 기반 게시글 필터링 기능, 실시간 채팅 기능을 통한 원활한 의사소통
